Java 模块化特性（jdk9最核心)

产生的意义

  1. java 运行环境的膨胀臃肿,每次JVM启动时候，至少加载30-60M的内存加载，主要是因为JVM需要加载rt.jar,不管
    其中的类是否被classLoader加载，（而模块化可以根据模块的需要加载程序运行需要的class)
  2. 当代码库大，创建复杂，盘根错节的增长，不同版本的类库交错依赖，导致让人头疼，阻碍了JAVA开发和运行效率。
  3. 很难真正的对代码进行封装，而系统并没有对不同部门之间依赖的明确概念，每个公共类都可以在其他类访问，导致无意并不想被公开访问的API
  4. 类路径本身存在问题， 你如果知晓所有需要的JAR都有了，或者是不是重复呢

模块化是什么

  1. 本质上，模块其实就是package 外再裹一层，也就是说用模块来管理各个package,通过申明某个package暴露。不声明就是隐藏。

好处有哪些

  1. 减少内存的使用
  2. 只需要必要模块，而非所有JDK模块，简化开发和维护
  3. 改进，使其适应不同大小的计算设备
  4. 改进安全，可维护性，提高性能

代码体现

Jshell

什么是Jshell

  1. REPL(read-evaluate-print-loop)像Python和Scala之类的语法，有了交互式的编程环境,开发者只需要输入一些代码，就可以编译前获得程序的反馈，而之前的Java必须执行代码，创建文件，声明类。
  2. 设计理念–即写即用

用法

cmd下直接输入jshell，就即可输入代码执行获取结果。（可以定以方法，变量，表达式等）

多版本兼容JAR包

意义何在

当出现了新版本的话，我们的库需要花费大量时间去匹配新版本，这既是意味着没有动力去切换版本，那么
Java9 可以兼容不同版本的jar包，让你可以使用java9新特性，并且可以依旧兼容你的jar包

语法层面改进

接口的私有方法

  1. jdk7中只能声明全局常量&抽象方法
  2. JDK8 中可以声明静态方法和默认实现方法
  3. JDK9 增加了可以定义接口私有方法 private void method(){};

钻石操作符的使用

我们能够与匿名实现类共同使用钻石操作符了；

XXX xx=new XXXX<>{};//jdk8中编译是通不过的

那么JDK9中编译通过，他的主要意思是：创建了一个继承XXXX的一个匿名实现类。

异常try结构的升级

下划线命名的限制

  1. Java8中可以独立使用标识符 如 String _=”hello”;
  2. java 9中规定“_”不再可以单独使用命名；否则编译不通过；

API层面变化

String底层结构的变化

Java9之前，String实现的底层是char[] 字符数组。char 使用了2个字节。这样会有内存的浪费。所以Java9变更了
String的底层结构为 byte[]数组。但是我们的中文使用了2个字节，也就是byte保持字母可以，但是中文则么办呢，
所以String底层还添加了Encode编码的概念，如果你使用utf编码，那么底层依旧是2个字节，而拉丁字符就使用1个字节，这样就可以达到节省内存的目的。（中文默认使用UTF-16，非中文使用IOS标准字符集）

创建只读集合(Collection)

传统的我们要创建一个只读不可变的集合，就需要创建实例，添加等操作。

增加Stream API

java9中的Stream接口中添加了4个新的方法：dropWhile,takeWhile,ofNullable,还有一个新增iterate方法的重载方法

提供了一个Predicate来指定什么时候结束迭代。

Option的变更

jdk9中给Option中添加了一个stream()方法，获取到stream，这样就可以将option中保持的数据进行Stream流操作了。

全新的HTTP请求客户端API

  1. 关于HTTP1/HTTP2 主要区别是如果在客户端和服务器端构建和传输数据，HTTP1.1依赖于请求/响应周期。HTTP/2允许服务器 push 数据，他可以发送比客户端请求更多的数据，使得他可以优先处理并发送对于首先加载网页的重要数据
  2. Java9中使用HTTPClient来替代HttpURLConnection ,并提供了webSocket和HTTP/2的支持。
  3. HTTP/2提供了流&服务器推送等功能。
