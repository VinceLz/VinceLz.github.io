cookie缓存

因为web基于java写的，所以java web有jcookie回话机制，用来保存当前session中的用户，但是安卓okhttp框架没有实现cookid的自动缓存，这样导致android一直处于无状态，那么web端也就无法实现对android用户的识别，期间想过使用token机制进行验证已检测身份，但是app的情况，最终还是选择了实现cookie缓存，还好okhttp框架比较强大，所以我们可以定制内部功能部分代码：

//将持久化的cookies缓存到内存中 即map cookies
   Map<String, ?> prefsMap = cookiePrefs.getAll();
   for (Map.Entry<String, ?> entry : prefsMap.entrySet()) {
       String[] cookieNames = TextUtils.split((String) entry.getValue(), ",");
       for (String name : cookieNames) {
           String encodedCookie = cookiePrefs.getString(name, null);
           if (encodedCookie != null) {
               Cookie decodedCookie = decodeCookie(encodedCookie);
               if (decodedCookie != null) {
                   if (!cookies.containsKey(entry.getKey())) {
                       cookies.put(entry.getKey(), new ConcurrentHashMap<String, Cookie>());
                   }
                   cookies.get(entry.getKey()).put(name, decodedCookie);
               }
           }
       }




    //将cookies持久化到本地
   SharedPreferences.Editor prefsWriter = cookiePrefs.edit();
   Log.v("#####SharedPr","SharedPreferences.cookie.save=");
   prefsWriter.putString(url.host(), TextUtils.join(",", cookies.get(url.host()).keySet()));
   prefsWriter.putString(name, encodeCookie(new SerializableOkHttpCookies(cookie)));
   prefsWriter.apply();

我们把cookie缓存到内存中，并最终实现持久化到本地，这样，我们后端就可以读取cookie，实现业务处理。

#通信框架#
其实我用过很多种通信框架，但是觉得OKHTTP框架是最好用的一款，所以建议大家也学习下okhttp框架的使用，对我们开发一套好的app至关重要。

实时推送

消息推送
1 长连接  心跳的骚扰  要求实时性高
2消息推送  实时要求不高  手机端  跑一个timer 10分钟发送一个数据udp  服务器接收到就推送
  1. 购买市面上的推送服务 —-资金问题不考虑
  2. 使用http请求 轮询 —-不够及时推送，并且加大服务器的负载放弃
  3. DDPush —-开源，免费 功能实现相对不错，

因为这种app不会大量推送消息，所以最终我们选择了DDpush这个开源的推送框架，并且在实现后，功能相对比较满意。

短信功能

俗话说现在的任务是无法脱离手机的，所以一款好的app必须要集成短信功能，所以最终选择了阿里大鱼短信服务平台，短信在android基本不需要什么实现，只需要加一个图形验证码就可以防止短信攻击，web后端有部分代码实现，我将会在另外一篇文章中介绍

#Frament#
Frament不得不说，确实比Activy开发高效的很多，我们也不需要管理那么多Activy，所以建议没有使用过的同学可以试试。

#安卓线程通信#
其实安卓线程通信的方式有很多，首先从系统架构上来说，我们可以使用信号量实现信息的传递，所以安卓框架已经为我们添加 了很多种线程之间的通信

  1. Hander 通信 一般采用的较多
  2. 文件通信 共享一个文件达到信息的交流
  3. MessageHander 这种通信比Hander难，但是同时他可以传递信息，
  4. ContentProvider 这种大家都应该用过吧
  5. Socket 这种一般用的很少

以上3种我自己使用过，其实实现通信的方法还有很多，大家可以google下

#UI 触摸滑动#
对于安卓的滑动触摸，其实里面的坑也有很多，尤其是比较复杂的程序，他需要对滑动检测的特别详细，一定要避免滑动冲突导致的问题，也需要对滑动事件的传播方式有所了解。
