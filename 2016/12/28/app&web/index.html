自定义JSON

public class JSON implements Serializable {

private static final long serialVersionUID = 1L;
private JSONObject json;

public JSON(JSONObject jsonObject) {
    jsonConfig.setJsonPropertyFilter(filter);
    this.json = jsonObject;
}

public void add(String key, Object value) {
    json.element(key, value, jsonConfig);
}

@Override
public String toString() {
    return json.toString();
}

JsonConfig jsonConfig = new JsonConfig();
PropertyFilter filter = new PropertyFilter() {
    public boolean apply(Object object, String fieldName, Object fieldValue) {
        return null == fieldValue || "".equals(fieldValue);
    }
};

      response.setStatus(HttpStatus.OK.value()); //设置状态码

#mybitis#
MyBatis/Ibatis中#和$的区别

  1. #将传入的数据都当成一个字符串，会对自动传入的数据加一个双引号。如：order by #user_id#，如果传入的值是111,那么解析成sql时的值为order by “111”, 如果传入的值是id，则解析成的sql为order by “id”.
  2. $将传入的数据直接显示生成在sql中。如：order by $user_id$，如果传入的值是111,那么解析成sql时的值为order by user_id, 如果传入的值是id，则解析成的sql为order by id.

  3. #方式能够很大程度防止sql注入。
    4.$方式无法防止Sql注入。
    5.$方式一般用于传入数据库对象，例如传入表名.
    6.一般能用#的就别用$.
    ps:在使用mybatis中还遇到<![CDATA[]]>的用法，在该符号内的语句，将不会被当成字符串来处理，而是直接当成sql语句，比如要执行一个存储过程。
    mybatis 映射文件中判断字符串是否为”” xxx!=’’ 和 xxx!=null 有区别

    POJO get/set

    java实体类pojo我们一般都需要提供get/set方法，但是有时候我们也可以把get/set灵活的运用起来，如何用呢？ 假如我们有一个值，对应的是数组，我们在数据库字段中要么一个值为一列，要么多个值使用特殊字符隔开，但是mybatis却不会对于的把查询到的值转换为数组的，那么我们就可以在get/set方法上面下功夫，其实框架不都是在Java反射的原理上进行的吗，反射最终访问属性都是通过get/set进行的，那么我们就可以把数据库中查询出来的字符串如 a,b,c,d,f,g,g 在java set方法中进行切分，然后赋值即可，实现如下

    public static String[] stringImage2Array(String img) {

进行检测验证码是否正确，并且对时间进行判断，是否超过10分钟。

public static boolean compTo(String currentTime, int number)
        throws ParseException {
    SimpleDateFormat myFmt1 = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

    Date current = new java.util.Date();
    Date parse = myFmt1.parse(currentTime);
    long time = parse.getTime();
    long time2 = current.getTime();
    if ((time2 - time) / 60000 <= number) {
        return true;
    } else {
        return false;
    }
}

#spring 异常#
@ExceptionHandler 这是spring 提供的在方法上面的异常，如我们检测某个方法如果发生某种特定的异常，才会拦截就需要使用这个注解，ExceptionHandler 注解 是在方法上的异常，当这个类中的任何方法发现异常就会执行注解方法，可以拦截指定的异常，如果不拦截，就会被上层拦截器异常拦截 （全局异常拦截）

#有序的property#

写一个类继承properties
重写一些方法
public class OrderedProperties extends Properties {
private static final long serialVersionUID = -4627607243846121965L;

private final LinkedHashSet<Object> keys = new LinkedHashSet<Object>();

public Enumeration<Object> keys() {
    return Collections.<Object> enumeration(keys);
}

public Object put(Object key, Object value) {
    keys.add(key);
    return super.put(key, value);
}

public Set<Object> keySet() {
    return keys;
}

public Set<String> stringPropertyNames() {
    Set<String> set = new LinkedHashSet<String>(); //内部使用LinkedHashSet 就可以实现有序

    for (Object key : this.keys) {
        set.add((String) key);
    }

    return set;
}

加密

对于加密，一般有对称算法AES，非对称算法RSA，MD5 ,BASE64 等，对于这个项目中，因为面临交易，所以必须对登录，交易进行加密，最终我们选择了非对称算法RSA ,即在用户请求登录的时候，去请求服务器，服务器返回公钥，服务器拥有私钥。最终，我们自客户端对密码进行加密，服务器最终解密，完成本次登录。其实对于安全问题，还有很大的学问，所以我们还需要学习，研究。
