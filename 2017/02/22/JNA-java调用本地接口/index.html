JNA java调用本地接口

Veröffentlicht am 2017-02-22

之前有个小项目需要做一个关于vpn的拨号的功能,google后发现java版的vpn拨号基本没有,所以办法了只能自己动手写一个关于java版的vpn工具类，所以特此介绍下关于java JNA

Java JNA?

什么是java jna呢？  JNA(java native access) 是一个开源的java框架，是SUN公司主导开发的一套建
立在JNI基础上的一个框架,它极大的简化了JNI方法，同时也非常强大！

java jni?

我们知道,使用JNI调用dll/so类库是非常的麻烦的，你得先使用SUN规定的数据类型来替代c语言的数据类型，然后才可以通过native 声明的本地方法来调
用dll中的公开方法,就这2部已经极大的加大了我们的开发难度。所以才会诞生了JNA

如何使用？

如何使用JNI 我们就在这里不多介绍了，毕竟有了比它更简单方便的框架了 

 1.下载相关的jar包
     可以去相关的官方下载jan.jar包 
 2.开始使用jna 

    java代码：

    public  class test{

        public interface vpn extends Library{

         vpn vpnHelp=Native.loadLibrary("rasapi32",vpn.class)            
         //上述代码表示我们要调用系统的rasapi32.dll的函数
        //以下开始声明我们所需要调用的函数
        int RasSetEntryPropertiesA(String lpszPhonebook,String lpszEntry,RASENTRY rasEntry,int dwSize,int DeviceInfo,int infoSzie);

        int RasDialA(int rasDal,String phoneBook,RASDIALPARAMS rs,int type,int fier,int rasConnn);
    } 
    //至此我们声明完毕，

    public static void main(String []arg){

        vpn.vpnHelp.RasSetEntryPropertiesA(); //创建vpn
        vpn.vpnHelp.RasDialA(); //拨号vpn

    }


}

简单吧，其实SUN开发的这个框架极大了简化了我们调用dll/so的成本，只需要我们继承Library,就可以加载
系统dll/第三方dll,即可实现调用dll,并且不需要担心类型不匹配，框架内部已经为我们实现了类型之间的转换。

如何选择？

那么我们如何选择JNA/JNI呢？ 如果我们对性能要求不是很高的话，使用JNA是最好的选择了，但是对性能要求的话，我建议大家使用JNI吧，毕竟他没有通过框架运作，几乎没有什么性能损失,同时面临着类型转换的难题,各有利弊吧！！
