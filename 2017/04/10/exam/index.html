慕课视频网站&在线答题实现

Posted on 2017-04-10

前言

继上次完成了在线预览文档的功能后，这次又需要实现一个在线答题的功能，目前来说我从来没有实现过此类功能，相比而言实现很简单，只不过是如何实现的比较好。

需求分析

总结分析我们发现，要实现一个答题功能，而且还需要记录成绩，简答题的答案功能。对于一套试题来说，包含很多试题，比如选择题、简答题、可能后续还需要编程题等等。那么试题还需要计时功能，总不得让学生无限期的去做一套题，综上分析，我们得出一个结论：

1. 试卷中包含若干试题 典型的一 对应 多的关系
2. 试卷需要一些附属的属性，试卷名、用时、类型等
3. 试题包含多种类型。选择题、多选题、简答题等

实现

数据库设计

经过上述分析，我们发现一共需要4张表

1. db_question 试题表，记录试题，包含题干、答案、说明、类型（多选，单选，简答）
2. db_paper 试卷表 与 试题表 属于一对多关系映射
3. db_exam 考试结果表，记录了学生答题情况，包括选择题的得分情况
4. db_exam_resume 简答题，对于简答题，我们无法给出直观的分数，需要教师去评分，所以我们需要多一张表来维护学生的答案。

代码实现

试卷展示

这个功能，只需要查询db_paper表选择出全部的试卷，交由前端显示

select * from db_paper;  //当然实际开发中不建议大家直接用*，哪怕是所有的字段都需要也别，这样会导致数据库浪费性能

答题

答题，就需要前端传给后端试卷的id，然后去数据库查询该试卷下的所有题型

select * from db_question where paperId=?

查询出来这个试题是一个集合list,然后我们需要遍历整个集合，拆分这个集合，将整个集合分为若干个几个，单项集合、多选项集合、简答题集合等等。这样，交由前端根据不同的类型选择不同的表现，如单项就需要radbutton按钮，多选就需要使用checkbox，简答题当然用textare，当然具体使用什么现实，有很多种。主要是前端需要负责一个计时功能，因为我们只是实现一个比较基础的在线答题功能，如果需要专业的话，计时功能不仅前端显示，后端也要计时，防止作弊，而且还需要试题恢复功能，也就是当用户答了若干道题，你就应该使用ajax向后台传递答案，而不是等他完成了全部的试题主动去触发提交，所以这个是根据项目所定的。

提交之后

当用户点击了提交按钮或者是计时结束自动提交后，后端就需要判断选择题的答题情况，在这之前我们的前端是这样设计的。

在题干上配置name="id-t-${试题id}" valus="试题id"   在选项上配置name="id-c-${试题id} values="A/B/C"

总结

其实实现一个在线做题的功能，比较简单，只不过我从没有做过相关需求，开始觉得难，但是其实实现起来，也就用了不到半天的功夫，所以同学们不管做什么事情，都要对自己有信心，可能结果想的比你要简单的多。上述就是我的在线试题实现的基本过程，如果有同学要实现类似功能，不乏提供一种设计思路吧，当然这种设计只是实现功能，从效率和性能上可能差的很远，但是项目需求不是很强烈，所以不需要太过纠结在这里。
