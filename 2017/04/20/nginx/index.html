nginx总结

Posted on 2017-04-20

nginx

nginx一款轻量级而又高性能的http和反向代理服务器，来自俄罗斯，目前阿里、腾讯、百度、搜狐等在用这款，足以说明它的性能和功能有多强大了。具体的描述大家百度一下吧。

nginx功能

反向代理

反向代理（Reverse Proxy）方式是指以代理服务器来接受internet上的连接请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的结果返回给internet上请求连接的客户端，此时代理服务器对外就表现为一个反向代理服务器。—摘自百度

根据百度我们得知，反向代理是这样的：

http请求--->nginx服务器--->后端服务器

反向代理用我们的大白话就是，我找一个代理人去请求数据，并返回给我。那么这么做有什么用呢？ 其实一般反向代理用在动静分离，也就是静态&动态页面。这也就说明当我们访问一个页面的时候，别看人家后缀是.html，就以为是静态页面，说不准，人家是通过nginx反向代理后还是去请求了后端服务器，这也叫做伪静态。

负载均衡

负载均衡，有什么用呢，当我们有这样一个场景：后端只有一台服务器在运行，但是前端大量用户访问，那么可以肯定的是这台服务器迟早支撑不住的，那么就需要我们扩展几台服务器，同时运行一个项目，那么问题就来了，前端去访问谁呢？如何访问才能让请求平均分配，而不是集中一台，不然问题又回到原点了。那么nginx就可以满足我们上述的所有需求。

                                后端服务器A
http请求---->nginx服务器          后端服务器B
                                后端服务器C

一些优化

我们不仅可以使用负载均衡和反向代理，还可以使用nginx对网站进行一些优化，那么应该下这几方面优化：

  1. 对于开发人员—-合并css, 背景图片, 减少mysql查询等.
  2. 对于运维 nginx的expires ,利用浏览器缓存等,减少查询.
  3. 利用cdn来响应请求
  4. 使用阿里的nginx一款插件，可以请求一次，下载多个文件。
