java版在线网盘系统

Posted on 2017-04-18

#前言#
本来这个系统是给实验室做的，并且任务也不属于的，是由一位学长开发的，但是由于学长实习去了，没有时间完成所以临时把项目源码给我了，但是由于学长电脑问题,sql数据库丢失，导致我基本没办法继续开发，所以当时决定重新开发一套小型web网盘系统，在此记录下自己的项目经历

#项目预览#

项目部分截图就是这些，还有很多具体的功能，没有一一截图，有兴趣的同学可以去我的git下载，整体网站是前端是使用bootstarp框架做的，模仿百度网盘的风格，在这里不得不提下，bootstarp确实很好用，对于我们这些搞后端的，对于前端简单的实现可以，稍微复杂的就有难度了，毕竟术业有专攻吧，所以，后端学学bootstarp框架也是很不错的，当你再缺少前端的时候，可以用他做一套很不错的界面。

数据库的设计

当时做这个系统的时候，自己还是一个不折不扣的菜鸟，对于设计数据库，确实很头疼，尤其是文件是以树的形式展现的，当时对 一对多 一对一 多对多的设计理念还是不太懂，所以就查阅各种设计文档，最终才凑合的设计完了数据库这一关，所以对于数据库，我们必须要掌握如下

  1. 一 对 一 举个例子，公民和身份证
  2. 多 对一 如 学生和某课程的老师
  3. 多 对 多 学生和所有的老师

#Linux文件管理#
文件管理，就是打开web网页，可以实现像本地文件管理一样的功能，当时设计采用的是linux 文件目录结构

/
 --bin
 --etc
 --sbin

例如像这种，当用户注册后，我们系统就会在数据库中为其添加一个根目录的id,数据库设计如下

cid,pid,cName,cDate,cF,isShare

我们每一个用户都有一个cid属性，表示用户的根目录，当用户登录后，后台根据cid来抓取他的树状目录，其中pid和cid是一一对应的，表示当前列的父级。

#Progress java #
当时有这样一个需求，需要把后端上传的目录进度实时显示到前端，其实设计起来很简单，但是实现起来却又点费力，之前自己动手写了一个定时器，每1/s 通过ajax向后端请求进度，完成后，效果不是很好，最终采用了java为我们提供的ProgressListener 这个监听器，他可以实现实时监听后台文件上传进度，效果比较满意。所以有兴趣的同学可以百度下 这个监听器，他是上传的fileupload jar中的。

#其他功能#
分享功能，例如百度网盘可以分享文件，文件夹等，具体实现起来其实也很简单，毕竟没有做到百度网盘那种高度，实现原理就是分享的如果是文件夹那么就把文件夹id作为参数，如果是文件也同理，百度网盘有密码功能，其实实现这个功能也很简单，只要在数据库中加加一张分享表，表中全是分享链接，附带密码功能即可

#总结#
其实今天看来，乃至在其他人眼里，这个网盘项目其实很简单，确实很简单，但是对于当时自己刚入门，要想实现这样的功能，确实有点难度，好在自己挺过来了，虽然不敢说现在的自己有多厉害，但是毕竟努力过…..
