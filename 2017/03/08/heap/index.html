前言

做完了堆的一些操作，我们来看关于图的一些知识，图，我们比较直观，就是用一些形状来简单化
我们所要抽象的物品，那么在数据结构中，图也是很重要的一种，同时图在我们实际生活中也用到很
多，譬如多个货运站之间的建立关系，如何做到路程最短等等…..那么跟我的脚步一起去看看，关于
图的一些基本性质我在这里就不过多解释了,我在这里就介绍一些关于图的一些应用算法。

拓扑排序

拓扑排序，如何理解呢？

这张图是我从<算法与数据结构>中截取的图，这张图表示美国高校的课程，如图，它有方向，也就是
我们所说的有向图,方向表示必须先选修完一门课程才可以根据方向选修下一门,那么就有人要问了,没有
方向的图就不在我们的讨论范围呢吗？当然,对于拓扑排序,没有不是有向图,我们还有研究的意义嘛,因为
无向图的方向是任意的,那么从理论上说任何一张排序都是拓扑排序,所以在这里我们限定在有向图。其实
我们要将的算法核心思想就是 先将整个图遍历一会，检测入度为0的结点，将它保存到一个队列中，当队列不为空的时候，就出队，将出队的结点删除，并与该结点相互连接的结点入度减少1，然后重复上述操作，继续检查是否有入度为0的，入队，出队，删除……..最终我们把删除的结点顺序排列，就是我们所要求的拓扑排序

广搜

广度搜索：广度，其实就是当我遍历到一个结点的时候，我会优先将该结点的所有子结点遍历完成后，才会遍历子结点下的子结点，不知道我这样说，大家是否明白呢？用图说明吧,需要说明一点深搜广搜我们一般研究的是有向图，毕竟无向图没有研究的价值。

如图，这是一些图组成的,根据广搜的性质，我们先从0号结点开始，当然不是必须从0开始，只是我们默认找一个结点，如图 广搜的顺序:0 -> 1 -> 2 -> 3 -> 4 -> 5 -> 6，你看，我用语言叙述一次，广搜，就是
要凸显出它的广度，我遍历完0结点后，他有子节点，1 2 那我就先把子节点全部遍历完成后，才去遍历子节点的子节点，这就是广度遍历。在这里，我贴出代码的大概实现：

深搜

深度搜索：其实凸显的是深度这个词，就是当我遍历完一个结点后，在去遍历它的第一个结点，以此类推，
直到不能进去深入的时候，我才会退出来，从下一个开始。如上图中的遍历顺序是：0 -> 1 -> 3 -> 4 ->
2 -> 5 -> 6

深搜与广搜的区别

深搜使用的是栈，广搜使用的是队列，深搜时间慢，但是省空间，深搜时间慢，但是占用空间多
深搜广搜一般用在无权有向图中。
