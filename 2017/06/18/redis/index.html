redis

Veröffentlicht am 2017-06-18

纲要

现在的状态就是每周写一个博文，记录下这周实习的内容，并对学到的东西进行总结，这周记录的就是redis、memcache,这次算不上实战，是在导师的要求下，实现一个缓存管理系统。

技术要点

  1. 要实现redis/memcache统一性，也就是说java层面上要屏蔽redis/memcache的差异
  2. 按系统分类管理并配置一定的权限
  3. 格式化显示缓存内容
  4. 对缓存系统进行预警
  5. 其他点的扩展

代码实现

屏蔽redis/memcache的差异

我们都知道，redis/memcache差异很大，首先他们的数据结构不同 ,redis提供string set list zset ，而memcache只提供了string 形式的存储格式。那么我们要屏蔽他们的差异，就需要解决他们在数据结构的差异。修改源代码肯定是行不通的，那么只有我们在代码层面上解决。故而采用了面向接口编程，

  • CacheOps 缓存操作接口 提供对缓存的增删改查等操作
  • CacheServiceBuild 缓存连接接口，解决redis/memcache连接方式不同
  • CacheFactory 工厂类，提供对redis/memcache的初始化

后续分析

这样我们在接口实现上完成了对redis和memcache的差异，那么在我们真正请求的时候，可以根据key计算出对于的服务器类型，如果是redis就初始一个redis的客户端，并进行实例rediOps操作类。

其实我一直有疑问，谁会没事把redis和memcache同时用在项目里，最后得知，其实在一些老的项目里，一直使用的是memcache,如果要修改成redis,那么可能在一定程度上要修改很多代码，这个代价是很昂贵的，所以我们必须兼容老系统。

通过上述代码实现了redis/memcache的差异屏蔽，但是效率可能最差的，所以在一定程度上还要优化，其实最大的优化部分就是如何使用memcache模拟出redis的数据结构。后续想想，肯定还会有更精妙的方法实现。
